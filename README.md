# Aquarius PI

Raspberry Pi 5 playout system. Drives an RTMP stream using mpv, Chromium, and ffmpeg.
To see how you configure programme lists and nmptv.json please refer to the [Aquarius readme.md](https://github.com/avrovulcanxh607/aquarius/blob/master/README.md).

## Requirements

- Raspberry Pi 5 running Debian (Bookworm/Trixie)
- PulseAudio
- Xvfb (virtual framebuffer)
- mpv
- chromium
- ffmpeg
- openbox

## Setup

```bash
chmod +x setup.sh
sudo ./setup.sh
```

This installs all dependencies, creates the `aquarius` PulseAudio sink, loads the `bcm2835-codec` kernel module, and sets up the systemd service.

## Running

### Full playout (scheduled)

```bash
python3 aquarius.py
```

Reads `command_output.json` (generated by `listings_creator.py`) and executes commands at scheduled times.

### Test mode

```bash
python3 aquarius.py --test                          # OS1 (RTMP stream)
python3 aquarius.py --test --source clock           # Clock website
python3 aquarius.py --test --source breakfiller     # Textbulletin
python3 aquarius.py --test --source ident           # Random ident MP4
python3 aquarius.py --test --source media           # Media file
```

### Stop everything

```bash
bash stop.sh
```

## Architecture

### Video pipeline

```
Sources (mpv/chromium) → Xvfb (:99) → ffmpeg → RTMP (libx264 ultrafast)
```

- **mpv** renders video to Xvfb via `--vo=x11 --hwdec=no`
- **Chromium** renders websites to Xvfb via `--ozone-platform=x11 --disable-gpu --kiosk`
- **ffmpeg** captures Xvfb + PulseAudio and encodes to RTMP

### Audio pipeline

```
Sources → PulseAudio (aquarius) → ffmpeg → RTMP (AAC)
```

- Null sink `aquarius` routes all audio
- ffmpeg captures from `aquarius.monitor`

### mpv instances

| Instance | Socket | Purpose |
|----------|--------|---------|
| os1 | `/tmp/aquarius-mpv-os1.sock` | RTMP input stream |
| player | `/tmp/aquarius-mpv-player.sock` | Ident + media files (single instance, switches via IPC `loadfile`) |

### Scene types

| Scene | Source | Notes |
|-------|--------|-------|
| OS 1 | mpv (os1) | RTMP stream input |
| Clock | Chromium | Website clock |
| Breakfiller | Chromium | Textbulletin website |
| Ident | mpv (player) | Random MP4 from ident folder |
| Media 1 | mpv (player) | Video file (loaded via LOAD command) |

### IPC communication

mpv uses JSON IPC over Unix sockets:
- `loadfile <url> replace` - switch to new file (no restart needed)
- `set_property pause false` - unpause
- `quit` - stop mpv

## Files

| File | Description |
|------|-------------|
| `aquarius.py` | Main playout controller |
| `aquarius_config.json` | Configuration (RTMP URL, website URLs, ident folder) |
| `listings_creator.py` | Schedule generator (nmptv.json → command_output.json) |
| `setup.sh` | Installation script |
| `stop.sh` | Kill all Aquarius processes |

## Config

`aquarius_config.json`:

```json
{
  "rtmp_url": "rtmp://server/app/stream",
  "os1_rtmp_url": "rtmp://source/stream",
  "clock_url": "https://intern.bollenstreekmedianetwerk.nl/klok2/",
  "breakfiller_url": "https://intern.bollenstreekmedianetwerk.nl/textbulletin2/test.php",
  "ident_folder": "/mnt/video/Fillers/bmntv2/idents/",
  "display": ":99",
  "resolution": "1920x1080",
  "framerate": 25
}
```
